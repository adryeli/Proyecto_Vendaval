## рҹҢӘпёҸ ВҝQuГ© es Proyecto Vendaval?

**Vendaval** es una aplicaciГіn de consola hecha en **Python** que monitoriza el tiempo en ciudades de EspaГұa.

Con ella puedes:

- рҹ“қ **Registrar** datos climГЎticos manualmente (temperatura, humedad, viento, lluvia)

- рҹ“Ҡ **Consultar** el histГіrico de tiempo y alertas filtrado por ciudad y fecha
- рҹ“Ҳ **Ver estadГӯsticas** en un dashboard grГЎfico interactivo
- рҹ”„ **Comparar** tus registros manuales con los datos reales de la API

Todo desde la terminal, con una interfaz colorida y fГЎcil de usar.

### Paso 1.5 вҖ” Crea tu rama de trabajo (branch)
 
> рҹ’Ў **ВҝQuГ© es una rama y por quГ© trabajar siempre en una?**
> Imagina que el proyecto es una casa. La rama `main` es la casa principal вҖ” siempre tiene que estar limpia y funcionando. Una **rama** es como construir una habitaciГіn nueva en un anexo separado. Trabajas ahГӯ sin tocar la casa principal. Cuando estГ© lista, la conectas. AsГӯ nunca rompes lo que ya funciona.
>
> вҡ пёҸ **Regla de oro: nunca trabajes directamente en `main`.**
 
Antes de crear tu rama, **sincronГӯzate con el equipo** para partir de lo mГЎs reciente:
```bash
git checkout main                  # asegГәrate de estar en main
git pull upstream main             # descarga lo Гәltimo del proyecto original
git push origin main               # actualiza tu fork tambiГ©n
```
 
Ahora crea tu rama con un nombre descriptivo:
```bash
# Ejemplos de buenos nombres:

git checkout -b fix/error-alertas-humedad     # correcciГіn de bug
git checkout -b refactor/menu-submenus        # reorganizaciГіn de cГіdigo
```
 
Verifica que estГЎs en tu rama nueva:
```bash
git branch
# Aparece tu rama con * delante:
# * feature/login-pantalla
#   main
```
 
вң… **A partir de aquГӯ, todo tu trabajo va en esta rama.**

Una vez tienes tu rama, sigue este flujo cada dГӯa:
 

Mensajes de commit вҖ” cГіmo escribirlos bien:

## рҹҸ—пёҸ Estructura del cГіdigo вҖ” explicada para principiantes

> ВҝQuieres entender cГіmo estГЎ organizado internamente? AquГӯ la explicaciГіn sencilla.

El proyecto sigue el principio de **separaciГіn de responsabilidades**: cada archivo hace **una sola cosa**. AsГӯ es mГЎs fГЎcil de entender, mantener y testear.

```
El usuario escribe algo en la terminal
         вҶ“
    cli/menu.py          вҶ’ recoge la opciГіn y llama al backend
         вҶ“
backend/api/ o core/     вҶ’ hace el trabajo (API, validar, alertas)
         вҶ“
backend/storage/         вҶ’ guarda o lee del JSON
         вҶ“
    resultado devuelto a menu.py
         вҶ“
cli/display_helpers.py   вҶ’ muestra el resultado bonito en pantalla
```

**No hay clases** вҖ” todo el cГіdigo usa **funciones simples**, lo que hace que sea mГЎs fГЎcil de seguir para quien estГЎ aprendiendo Python.
